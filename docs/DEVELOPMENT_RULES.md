# Development Rules

## Общие правила для Codex
- Не создавать новые стадии без явного запроса.
- Не создавать новые статусы партии без явного запроса.
- Не ломать обратную совместимость данных и отчетов.
- Не менять формат `report.json` без необходимости.
- Сначала использовать уже существующие сущности и поля.
- Не вводить новые модели без обоснования.
- Документы в `docs/` считать источником истины для следующих задач.

## Правила работы с кодом
- Не делать рефакторинг ради рефакторинга.
- Не переписывать бизнес-логику, если задача про документацию или точечную корректировку.
- Не менять UI без отдельного запроса.
- Не трогать структуру проекта, если это не требуется задачей.

## Правила по домену
- Не выдумывать стадии, которых нет в `src/domain/constants.js`.
- Не выдумывать операции, которых нет в `src/domain/statusOperations.js`, `src/domain/batch.js`, `src/domain/journal.js` и связанных модулях.
- Не добавлять отдельную сущность `Location`, если в коде она представлена полями карточки и операции.
- Не добавлять отдельную сущность `Photo`, если в коде используются только URI и файлы экспорта.
- Не добавлять новый `User`-контур, если достаточно `currentUser` и профиля сотрудника.

## Правила по данным
- Любой новый ключ, который может попасть в отчет, должен быть проверен на совместимость.
- Если поле уже используется в журнале, его нельзя тихо переименовывать.
- Если значение вычисляется из операций, не нужно дублировать его как источник истины без причины.

## Правила по будущей админке
- Считать `report.json` контрактом.
- Не предполагать серверные таблицы, которых нет в приложении.
- Если админке нужен новый атрибут, сначала проверить, можно ли получить его из существующих данных мобильного приложения.

## Как действовать в сомнительных случаях
- Если сущность есть только в одном месте кода, описывать ее как реализацию, а не как отдельную доменную модель.
- Если правило видно только косвенно через вычисление, явно помечать его как вычисляемое.
- Если поведение не реализовано, писать: "Запланировано, но пока не реализовано".
